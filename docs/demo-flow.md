# Demo Flow

## 0. Environment

Local development uses `.env`.

Required local settings:

```text
EMBEDDING_PROVIDER=local
LOCAL_EMBEDDING_MODEL=D:\AI创业\AI模型\embedding-models\BAAI\bge-small-zh-v1.5
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=qwen2.5:3b
```

GitHub CI should use fake providers and must not depend on local model files or a running Ollama service.

## 1. Start Ollama

Start the Ollama service before testing `/answer` in local Ollama mode.

Terminal A:

```powershell
D:\ollama\ollama.exe serve
```

Terminal B:

```powershell
D:\ollama\ollama.exe list
```

Expected model:

```text
qwen2.5:3b
```

Optional manual chat check:

```powershell
D:\ollama\ollama.exe run qwen2.5:3b
```

## 3. Health Check

Call:

```text
GET /health
```

Expected:

```json
{"status": "ok"}
```

## 4. Upload

Call:

```text
POST /upload
```

Upload `hello.txt`:

```text
RAG uses retrieved context.
```

Copy returned `document_id`.

## 5. Retrieve

Call:

```text
POST /retrieve
```

Body:

```json
{
    "document_id": "doc_xxx",
    "question": "What does RAG use?",
    "top_k": 3
}
```

Check that `matches` contains relevant chunks from the uploaded document.

## 6. Answer

Call:

```text
POST /answer
```

Body:

```json
{
    "document_id": "doc_xxx",
    "question": "What does RAG use?",
    "top_k": 3
}
```

Check:

- `answer`: should be generated by the configured LLM provider.
- `provider`: should be `ollama:qwen2.5:3b` in local Ollama mode.
- `sources`: should contain retrieved chunks.
- `confidence_notes`: should explain whether the answer is grounded in retrieved context.

## 7. Docker Demo

Docker mode is mainly for API packaging verification.

```powershell
docker build -t personal-knowledge-rag .
docker run --rm -p 8000:8000 personal-knowledge-rag
```

Open:

```text
http://127.0.0.1:8000/docs
```

Docker does not automatically access Windows local model paths or the host Ollama service. Use fake providers for simple Docker verification unless volume mounts and host networking are configured.

For CI or simple Docker verification, use fake providers:

```text
EMBEDDING_PROVIDER=fake
LLM_PROVIDER=fake
```

## 8. CI And Local Dependency Notes

CI tests should verify application logic, provider selection, API behavior, and mocked provider behavior.

CI tests should not require:

- Windows local model paths.
- A downloaded embedding model.
- A running Ollama service.
- Private `.env` values.

Tests that truly load local models or call a real local Ollama service should be marked separately and skipped in CI.
