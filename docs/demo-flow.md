# Demo Flow

0. Configure the environment variables.
1. Start API
2. Upload sample document through Swagger
3. Retrieve relevant chunks with top_k=3
4. Check rerank metadata: score, rerank_score, candidate_rank
5. Call /answer and inspect answer, provider, sources, confidence_notes
6. Explain current limitations.

## 0. Environment

Local development uses `.env`.

Required local settings:

```text
EMBEDDING_PROVIDER=local
LOCAL_EMBEDDING_MODEL=D:\models\embedding-models\BAAI\bge-small-zh-v1.5

RERANKER_PROVIDER=cross_encoder
RERANKER_MODEL=D:\models\rerank-model\BAAI\bge-reranker-base

VECTOR_STORE_PROVIDER=sqlite
QDRANT_URL=http://127.0.0.1:6333
QDRANT_COLLECTION=personal_knowledge_chunks

LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://127.0.0.1:11434
OLLAMA_MODEL=qwen2.5:3b
```

GitHub CI should use fake providers and must not depend on local model files or a running Ollama service.

Call:

```text

```

Expected:

```json

```

## 4. Upload

Call:

```text
POST /upload
```

Upload a `.txt`, `.md`, `.csv`, `.pdf`, or `.docx` file.

Expected checks:

- Response includes `document_id`.
- Response includes `chunk_count`.
- Uploaded content is parsed into text preview.
- PDF and DOCX support is limited to machine-readable text; scanned PDF OCR is future work.

## 5. Retrieve

Call:

```text
POST /retrieve
```

Body:

```json
{
  "document_id": "doc_xxx",
  "question": "What does the uploaded document say?",
  "top_k": 3
}
```

Expected checks:

- `matches` contains relevant chunks from the uploaded document.
- Each match contains source text and similarity score.
- Retrieval quality should be inspected before trusting `/answer`.

## 6. Answer

Call:

```text
POST /answer
```

Body:

```json
{
  "document_id": "doc_xxx",
  "question": "What does the uploaded document say?",
  "top_k": 3
}
```

Expected checks:

- `answer`: generated by the configured LLM provider.
- `provider`: `ollama:qwen2.5:3b` in local Ollama mode.
- `sources`: retrieved chunks used as answer evidence.
- `confidence_notes`: explanation of whether the answer is grounded in retrieved context.

## 7. Docker Demo

Docker mode is mainly for API packaging verification.

```powershell
docker build -t personal-knowledge-rag .
docker run --rm -p 8000:8000 personal-knowledge-rag
```

Open:

```text
http://127.0.0.1:8000/docs
```

Docker does not automatically access Windows local model paths or the host Ollama service. Use fake providers for simple Docker verification unless volume mounts and host networking are configured.

For CI or simple Docker verification, use fake providers:

```text
EMBEDDING_PROVIDER=fake
RERANKER_PROVIDER=keyword
LLM_PROVIDER=fake
VECTOR_STORE_PROVIDER=sqlite
```

## 8. Current Limitations

- The default local path uses SQLite-backed vector storage, while Qdrant is available behind the VectorStore boundary.
- Retrieval evaluation is small and anchor-based, with metrics such as Hit Rate@K, Recall@K, and MRR.
- PDF and Word text extraction is supported for machine-readable documents; OCR for scanned PDFs is not implemented.
- The project is single-user and does not include authentication or user-level data isolation.
- Local LLM performance depends on laptop hardware.


## 9. CI And Local Dependency Notes

CI tests should verify application logic, provider selection, API behavior, and mocked provider behavior.

CI tests should not require:

- Windows local model paths.
- A downloaded embedding model.
- A running Ollama service.
- Private `.env` values.

Tests that truly load local models or call a real local Ollama service should be marked separately and skipped in CI.
